这里解释下部分 Hexo 命令的意思，详细指令请参考指令文档。

  • init - 新建一个网站
  • new - 新建一篇文章

  • deploy - 部署网站，命令可简写为 hexo d
  • clean - 清除缓存文件和已生成的静态文件

配置主题

配置主题文件

想要配置 NexT 的主题，传统的方法是直接修改相关配置文件 _config.yml or themes/next/_config.yml，但这种方式有一定弊端，当通过
git 或者 npm 升级 NexT 主题时，这些配置文件就会被覆盖，给主题配置带来不便。所以，NexT
现在鼓励用户使用 Alternate Theme Config，这是 Hexo
的一个特性，参考文档 Hexo Configuration。
使用其他配置文件时可通过命令 --config 来指定自定义配置文件路径：

使用 _config.[name].yml

将所有的主题配置集中于文件 _config.[name].yml 中，并替换站点配置文件 _config.yml 中属性 theme 的值为 [name]。

  1. 使用Hexo 5.0 及以上版本。
  2. 在站点根目录创建配置文件，如 _config.next.yml
  3. 复制主题配置文件 themes/next/_config.yml 的内容至文件 _config.next.yml。

使用 theme_config

所有的主题配置集中于站点配置文件 _config.yml 中，修改站点配置文件 _config.yml 中属性 theme 的值为
next，添加属性 theme_config，并将主题配置文件 themes/next/_config.yml 作为该属性的值，记得保持缩进。

页面美化

主题选择

目前 NexT 支持四种主题方案，分别是：

  • Muse - 默认主题，使用黑白色调，看起来较为干净。
  • Mist - 更紧凑的 Muse 版本，有整齐的单列视图。
  • Pisces - 双列主题，比较清新。
  • Gemini - 与 Pisces 有些相似，但有带阴影的柱块，对视觉更为敏感。

在配置文件中可选择你喜欢的主题(●’◡’●)。
